Package management¶

Much of the power of Python comes from the external libraries and packages. Unfortunately package management is a touchy subject. Currently, for the scientific stack, I recommend using the Anaconda distribution, which is designed to take care of the various versions of packages such that they are consistent and relatively interoperable. However, even so, you will often need to install packages. You can do this using the conda program.

The more native means is to use the pip program to install packages individually.

Finally, more recently, Windows have begun distributing Python with its OS. You can install directly from the Windows app store (although this is currently in a period of rapid development).

The versions of packages I am currently using for this demonstration are:
